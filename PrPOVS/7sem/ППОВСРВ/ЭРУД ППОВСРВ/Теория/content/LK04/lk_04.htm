Технология создания программ на языке С/С++

Стандарт языка ANSI С/С++. Основные элементы, типы данных, конструкции языка С/С++

Сложность современного программного обеспечения требует для его создания использования новых технологий программирования. Сложность внутреннего устройства современных процессоров цифровой обработки сигналов определяет серьезные требования к компетенции программистов. Большинство программистов слабо разбираются в архитектуре проблемно-ориентированных процессоров. Для обеспечения разработки большого количества программ требуется привлечение для разработки модулей программного обеспечения, не играющих существенной роли в обеспечении основного технологического процесса, программистов пишущих программы на языках высокого уровня, в частности С/С++. Для обеспечения результативности процесса программирования рекомендуется соблюдать стандарты языков.

Первый стандарт языка C был опубликован американским институтом ANSI. Через некоторое время он был принят международной организацией по стандартизации ISO, продолжившей выпускать следующие версии стандарта, которые стали приниматься как стандарт и институтом ANSI.
C89. В 1983 году Американский национальный институт стандартов сформировал комитет X3J11 для создания спецификации стандарта C. В 1989 году, после длительного и напряжённого процесса, стандарт был завершён и утверждён как ANSI X3.159-1989 «Язык программирования C». Именно эту версию часто называют «ANSI C», или «C89».
C90. В 1990 году, стандарт ANSI C (с небольшими изменениями) был принят Международной организацией по стандартизации (ISO) как ISO/IEC 9899:1990. Эту версию иногда называют C90. Однако, термины C89 и C90 относятся в сущности к одному языку.
C99. В марте 2000 года ANSI принял стандарт ISO/IEC 9899:1999. Этот стандарт обычно называют C99. Это и есть современный стандарт языка программирования C.
ANSI C поддерживается практически всеми распространёнными компиляторами языка C. Любая программа, написанная только с использованием стандарта и не допускающая специфических аппаратных допущений и использования сторонних библиотек, гарантированно должна работать на любой платформе с достаточно стандартной реализацией языка C.

Далее устраняем ошибки как на этапе компиляции, так и на этапе построения (компоновки) проекта. Обязательно обращаем внимание на предупреждения, которые генерирует система. Так предупреждение о отсутствии точки входа на этапе компоновки является серьезным предупреждением и требует коррекции:

----------------------------------------------------------------------------------------------------------------------------------------------

<Linking>
>> warning entry point symbol _c_int00 undefined

----------------------------------------------------------------------------------------------------------------------------------------------

означает, что в проекте не определена точка входа по сбросу питания (вектор прерывания 0)

Для устранения данной ошибки, если не подключена библиотека rts.lib или rts_ext.lib, необходимо определить точку входа следующим образом:

                        extern              void     c_int00            (void)  {
                        int        a,b,c;
                        a = 34;
                        b = 55;
                        c = b;
                        a += b;
                        b *= c;
                        c = 0;
                        return;
                        }

Выполняем построение проекта <Project><Rebuild All>. Предупреждение, появившееся в окне вывода сообщает об использовании вызова компоновщика по умолчанию, согласно профилю Debug . В результате этого действия получаем проект в виде выходного файла, который может быть загружен либо в симулятор, либо в процессор.
